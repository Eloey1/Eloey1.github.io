GRAPHICS SHOWCASE

DX11 PBR Renderer

C++ • DirectX 11 • HLSL • Compute Shaders

Architecture Overview

This is a forward+ renderer built to handle dynamic lighting with physical accuracy. It utilizes the Cook-Torrance BRDF model for lighting calculations and implements Image Based Lighting (IBL) for ambient terms.

PBR Lighting

Implemented Albedo, Normal, Metallic, and Roughness workflow. Includes HDR environment maps for realistic reflections.

Multi-Threading

Command list generation is split across worker threads to minimize CPU bottlenecks during the render pass.

Shadow Mapping

Cascaded Shadow Maps (CSM) to handle directional light shadows with PCF filtering for soft edges.

Implementation Details

1. The Render Loop

To decouple logic from rendering, I implemented a strict separation where the game logic pushes `RenderCommands` to a queue. This queue is sorted by material and depth to minimize state changes.

// Example: Submit function void Renderer::Submit(const Mesh& mesh, const Material& mat, const Transform& trans) { RenderCommand cmd; cmd.mesh = &mesh; cmd.material = &mat; cmd.transform = trans; // Sort key generation based on material ID and depth cmd.sortKey = CalculateSortKey(mat, trans); m_CommandQueue.push_back(cmd); }

2. Compute Shader Culling

To handle high object counts, I moved frustum culling to the GPU. A compute shader iterates over instance data buffers and produces an `IndirectArgs` buffer for `DrawInstancedIndirect`.
